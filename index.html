01 — Abstract

Compress structure.
Reject noise.

The paper proposes integrating a low-rank compression decision model into the operating system memory manager. Instead of treating every memory page equally, the OS samples a page, reshapes it into a matrix, and tests whether a compact SVD representation can preserve it with acceptable reconstruction error.

When RAM pressure is low, the system keeps more singular components for accuracy. As pressure rises, the selected rank becomes more aggressive — saving space on structured pages while filtering out random pages that would reconstruct poorly.

02 — Methodology

How SVD compression
works in the OS

01

Page Sampling

The OS samples an incoming memory page from the swap candidate list and converts it into a 2D matrix representation.

02

SVD Decomposition

Truncated SVD decomposes the matrix into U, Σ, and V components. Only the top-k singular values are retained.

03

Error Evaluation

Reconstruction error is compared against a dynamic threshold that tightens under high memory pressure.

04

Compress or Reject

Structured pages are compressed and stored. Random pages are stored raw — avoiding costly reconstruction errors.
